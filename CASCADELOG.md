# CASCADELOG

- Цель (критерии успеха): Подготовить интерактивное демо «Эсеп» к Customer Discovery с корректным учетом продаж и инвентаризации
- Ограничения/допущения: Vanilla JS, HTML, CSS, localStorage; демо не предназначено для реальной эксплуатации
- Ключевые решения:
  - Блокировать продажу, если хотя бы одного ингредиента недостаточно по рецепту
  - Фиксировать расчетные остатки в snapshot на старте инвентаризации
  - Рассчитывать расхождения и дашборд только по snapshot завершенной инвентаризации
- Статус: Этап 1 завершен
- Сделано:

  - Добавлен snapshot расчетных остатков на старте инвентаризации
  - Расчеты инвентаризации и дашборда переведены на зафиксированный snapshot
  - Новые продажи больше не искажают завершенную аналитику расхождений
