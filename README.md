# Эсеп

Касса и складской учёт для кофеен Кыргызстана, которыми **реально пользуются**.
Отдельный проект (не связан с другими репозиториями). Хакатон, MVP за 1 неделю.

> **Герой продукта — детектор утечки:** продал → склад списался сам → сигнал закупки → инвентаризация → «вот куда делись деньги» в сомах.

## Структура

```
docs/
  concept.md      · концепт «ДО»: проблема, ЦА, рынок (с источниками), бизнес-модель, Q&A
  pitch-deck.md   · сценарий выступления (10 слайдов + тайминг + реплики)
  brand.md        · бренд-система: палитра, типографика, UI-стиль
brand/
  logo.svg        · лого «эсеп.» (иконка + вордмарк)
  favicon.svg     · иконка приложения
deck/
  index.html      · питч-дек (открывается в браузере, навигация ← →)
app/
  index.html      · рабочий MVP-демо (касса → склад → инвентаризация → дашборд)
```

## Как запустить

Локальный сервер (нужен, потому что браузер блокирует `file://`):

```bash
cd kassabar
python3 -m http.server 8791
```

- Питч-дек → http://localhost:8791/deck/index.html · навигация: `←` / `→` / `Space`
- Приложение → http://localhost:8791/app/index.html

## Демо-сценарий (90 сек)

1. **Касса** — тапни несколько напитков (Латте, Капучино, Раф). Продажи фиксируются, ингредиенты списываются сами.
2. **Склад** — молоко упало само; горит «🔴 Пора заказать». Ничего не вводили руками.
3. **Инвентаризация** — «Смоделировать недостачу» → «Посчитать расхождение».
4. **Дашборд** — герой-цифра утечки в сомах + разбивка «где утекло». Кульминация: *«Вот куда делись деньги. Больше не на глаз.»*

Кнопка **«Сбросить демо»** возвращает начальные данные. Состояние хранится в localStorage браузера.
