# Эсеп

Касса и складской учёт для кофеен Кыргызстана, которыми **реально пользуются**.
Отдельный проект (не связан с другими репозиториями). Хакатон, MVP за 1 неделю.

> **Герой продукта — детектор утечки:** продал → склад списался сам → сигнал закупки → инвентаризация → «вот куда делись деньги» в сомах.

## Структура

```
docs/
  concept.md      · концепт «ДО»: проблема, ЦА, рынок (с источниками), бизнес-модель, Q&A
  pitch-deck.md   · сценарий выступления (10 слайдов + тайминг + реплики)
  brand.md        · бренд-система: палитра, типографика, UI-стиль
brand/
  logo.svg        · лого «эсеп.» (иконка + вордмарк)
  favicon.svg     · иконка приложения
deck/
  index.html      · питч-дек (открывается в браузере, навигация ← →)
app/
  index.html      · разметка рабочего MVP
  styles.css      · адаптивный интерфейс
  app.js          · пользовательские сценарии и хранение состояния
  domain.js       · проверяемая логика учёта и расчётов
tests/
  domain.test.js  · автоматические проверки учёта
```

## Как запустить

Локальный сервер (нужен, потому что браузер блокирует `file://`):

```bash
cd kassabar
python3 -m http.server 8791
```

- Питч-дек → http://localhost:8791/deck/index.html · навигация: `←` / `→` / `Space`
- Приложение → http://localhost:8791/app/index.html
- Публичное демо → https://startup-lime-pi.vercel.app/app/index.html

Проверка бизнес-логики:

```bash
npm test
```

## Демо-сценарий (90 сек)

1. **Касса** — продай 3 латте, 2 капучино и 1 раф: 6 чашек, 970 сомов выручки.
2. **Склад** — молоко упало 2 500 → 1 450 мл и пересекло порог 1 500 мл.
3. **Инвентаризация** — «Начать» → «Смоделировать недостачу» → «Посчитать расхождение».
4. **Дашборд** — 259 сомов COGS, 27% food cost и около 22 сомов динамической утечки.

Кнопка **«Сбросить демо»** возвращает начальные данные. Состояние хранится в localStorage браузера.

## Позиционирование

iiko и Poster дали широкий набор функций. Esep проверяет более короткий сценарий:
автосписание ингредиентов + инвентаризация + расхождение в сомах. Кыргызский язык, ГНС, WhatsApp и облачная синхронизация находятся в Roadmap.
